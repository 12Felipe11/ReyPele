Carpeta Avance

Sistema de Control de Estadio Inteligente

Java + Arduino (HC-SR04 + OLED SSD1306)

Laboratorio 1 - Avance de proyecto

¿Que es Avance?

Modulo Java que se comunica con un Arduino via puerto serial para leer sensores, controlar actuadores y exponer el estado del estadio.

Objetivo

Integrar hardware real (Arduino) con una aplicacion Java por capas.

Entradas

Lectura de distancia, presencia, conteo de personas e intensidad de luz.

Salidas

Consola interactiva, dashboard web en :8080 y pantalla OLED en Arduino.

Hardware

  • Arduino Uno R3 - microcontrolador ATmega328p.
  • HC-SR04 - sensor ultrasonico (distancia / presencia).
  • OLED SSD1306 128x64 - pantalla local de estado.
  • LED integrado como indicador de fallo de pantalla.

Pines

TRIG_PIN = 9
ECHO_PIN = 8
I2C OLED  = A4/A5 (SDA/SCL)

Arquitectura por capas

Presentacion

Consola (ConsoleUI) + dashboard web (WebDashboardServer en :8080).

Aplicacion

StadiumController interpreta comandos: READ, LIGHT, ALARM, MODE, SET, STATUS, HELP, EXIT.

Dominio

StadiumFacade orquesta sensores, actuadores, config y estrategias de modo.

Infraestructura

IHardwareComm + adaptadores: Arduino real (serial) o simulador.

Capa de Dominio

Sensores

  • DistanceSensor - distancia en cm + estado de presencia.
  • EntryCounterSensor - contador de personas en la puerta.

Actuadores

  • LightingZoneActuator - intensidad por zona (0-100%).
  • AlarmActuator - activa/desactiva alarma.

Modelo comun

  • SensorData parsea la trama DATA:count,dist,pres,light.
  • StadiumConfig guarda umbral de ocupacion y de distancia.

Capa de Aplicacion

StadiumController es el cerebro de comandos. Convierte texto del operador en llamadas a la fachada.

READ                       -> facade.readAllSensors()
LIGHT <0-100|ON|OFF>       -> facade.setLight(i)
ALARM ON|OFF               -> facade.setAlarm(bool)
MODE AUTO|MANUAL|EMERGENCY -> facade.changeMode(strategy)
SET THRESHOLD <personas>
SET DISTANCE  <cm>
STATUS / HELP / EXIT

Cada comando respeta el modo activo (por ejemplo, MANUAL bloquea automatizaciones).

Capa de Infraestructura

IHardwareComm

Interfaz comun: connect, readSensors, setAlarm, setLight, setThreshold.

ArduinoSerialAdapter

Comunicacion real con Arduino via jSerialComm a 9600 baud. Timeout 3s, reset 2s.

SimulatorAdapter

Genera datos sinteticos para probar sin hardware (HU-14).

ArduinoPortScanner

Autodeteccion de puerto (COMx, /dev/ttyACM*, /dev/ttyUSB*).

Capa de Presentacion

  • ConsoleUI - loop interactivo por teclado.
  • WebDashboardServer - servidor HTTP embebido en puerto 8080, se abre en el navegador al iniciar.
  • DashboardHtml - plantilla HTML del tablero con el estado en vivo.

El usuario ve dos vistas simultaneas: consola + tablero web.

Firmware Arduino (stadium_sensors.ino)

  • Mide distancia con HC-SR04 cada ciclo.
  • Detecta "paso" cuando el objeto se acerca y luego se aleja (bloqueo 800 ms anti-rebote).
  • Emite evento ENTRY:n en cada conteo nuevo.
  • Envia telemetria DATA:... cada 500 ms.
  • Actualiza pantalla OLED con distancia, contador y estado de actuadores.

Protocolo serial (9600 baud)

Arduino -> Java

READY                              // handshake de arranque
DATA:count,distance,presence,light // telemetria cada 500ms
ENTRY:totalCount                   // evento por paso detectado

Java -> Arduino

READ                  // pide lectura inmediata
ALARM_ON / ALARM_OFF
LIGHT_SET:0-100
SET_THRESHOLD:cm      // umbral del sensor de distancia

Patrones de diseno aplicados

Facade

StadiumFacade oculta la complejidad de sensores/actuadores/modos.

Strategy

IStadiumModeStrategy con variantes Auto, Manual y Emergency.

Factory

DeviceFactory -> RealHardwareFactory o SimulatedHardwareFactory.

Adapter

ArduinoSerialAdapter adapta jSerialComm a IHardwareComm.

Observer (informal)

Eventos ENTRY: propagan cambios desde el Arduino.

MVC por capas

Presentacion / Aplicacion / Dominio / Infraestructura.

Flujo al iniciar

  1. Main pregunta: 1) Simulador o 2) Arduino.
  2. Si es Arduino, ArduinoPortScanner detecta el puerto.
  3. La fabrica crea el IHardwareComm correspondiente.
  4. Se instancia StadiumFacade + StadiumController.
  5. Arranca WebDashboardServer en http://localhost:8080.
  6. Arranca ConsoleUI - loop de comandos hasta EXIT.

Como ejecutar

Compilar

javac -cp "Avance/lib/jSerialComm-2.11.0.jar" \
      -d Avance/out $(find Avance/src -name "*.java")

Ejecutar

java -cp "Avance/out:Avance/lib/jSerialComm-2.11.0.jar" Main

Subir firmware

Arduino IDE -> abrir Avance/arduino/stadium_sensors/stadium_sensors.ino
             -> seleccionar Uno -> Upload
